# Feature Folder

This folder contains all artifacts for one feature. Copy this template into `features/[feature-name]/` when starting a new feature.

---

## Structure

```
features/[feature-name]/
  README.md              ← this file
  initial-spec.md        ← you write this with spec-coach before running anything
  docs/
    dag.md               ← generated by /generate-dag
    state.md             ← generated by /generate-dag, updated by /next-step
    specs/
      [step-id].md       ← generated by /next-step, one per step
    tests/
      [step-id].feature  ← generated by /next-step, one per step
```

---

## Workflow

**1. Write the spec**
Use spec-coach to write `initial-spec.md` in Deictic Statute format before doing anything else. Add any reference docs this feature needs (API docs, migration notes, etc.) to this folder.

**2. Generate the plan**
Run `/generate-dag` pointed at this feature folder. Review the generated `docs/dag.md` and complete the prereqs checklist before proceeding.

**3. Build step by step**
Run `/next-step` for each step in the DAG. Each run generates a milestone spec and tests, builds, and opens a PR. Merge the PR before running the next step.

**4. Done**
When all steps are complete and all PRs are merged, this feature is done. Start the next feature by copying this template again.

---

## What you own

| File | Who creates it | When |
|------|---------------|------|
| `initial-spec.md` | You | Before running anything |
| Reference docs | You | Before running `/generate-dag` |
| `docs/dag.md` | `/generate-dag` | On first run |
| `docs/state.md` | `/generate-dag` | On first run |
| `docs/specs/[id].md` | `/next-step` | Per step |
| `docs/tests/[id].feature` | `/next-step` | Per step |

---

## Notes

- Do not edit `docs/state.md` by hand unless recovering from a failed build
- If a step is marked `in_progress` and needs to restart, change its status back to `pending` in `docs/dag.md` and clear `in_progress` in `docs/state.md`
- Reference docs added to this folder are automatically picked up by `/generate-dag` and `/next-step`
